## 🎨 設計系統：極簡現代劇場風格

### 色彩計畫

```
--primary-black: #0A0A0A        /* 主背景 — 深邃舞台 */
--primary-white: #F5F5F0        /* 主文字 — 聚光燈下的白 */
--accent-gray: #2A2A2A          /* 卡片/區塊背景 */
--mid-gray: #888888             /* 次要文字 */
--highlight: #FFFFFF            /* 強調元素 — 純白聚光 */
--subtle-warm: #C8B8A2          /* 點綴色 — 咖啡的溫度 */
--divider: #333333              /* 分隔線 */
```

### 字體系統

```
標題字體: "Playfair Display"（英文）/ "Noto Serif TC"（中文）— 襯線、劇場感
內文字體: "Inter"（英文）/ "Noto Sans TC"（中文）— 簡潔易讀
特殊裝飾: "Cormorant Garamond" — 用於 slogan 或引言
字級系統: 採用 clamp() 實現流體排版
```

### 視覺語言

- **大量留白**：區塊間距寬裕，內容呼吸感強
- **強對比排版**：大標題 + 細內文，粗細對比明顯
- **幾何舞台元素**：
  - 細線框矩形作為裝飾邊界（像舞台框架）
  - 圓形元素暗示聚光燈
  - 垂直/水平細線條作為段落分隔
- **微動效**：
  - 滾動時內容淡入（Intersection Observer）
  - 商品卡片 hover 微放大 + 陰影變化
  - 聚光燈追蹤效果（hero 區，滑鼠移動時光暈跟隨）
- **劇場隱喻**：
  - 頁面頂部可有「幕布拉開」的進場動畫
  - 區塊之間用「幕間」(Intermission) 概念過渡
  - 用「Act 1, Act 2...」作為隱性的區塊結構概念（不一定要明顯顯示）

## 🏗️ 網站結構與區塊規格

請按以下順序開發，每個區塊都是獨立的 `<section>`：

### Section 1: Hero 主視覺區

- 全螢幕高度 (100vh)
- 品牌名稱「咖啡小劇場」大字居中展示
- 副標語（例：「每一杯，都是一場獨白」）
- 背景：純黑 + 中央圓形漸層光暈模擬聚光燈
- 底部有向下滾動的提示箭頭（帶呼吸動畫）
- 選用：幕布緩緩拉開的 CSS 進場動畫

### Section 2: 品牌故事區

- 標題：品牌理念
- 左右分欄佈局（桌面）/ 上下堆疊（手機）
- 左側：2-3 段品牌故事文字（預留內容區，先以假文填充）
- 右側：一張佔位圖片（劇場/咖啡意象）
- 設計元素：細線矩形框包裹文字區域，營造「劇本頁面」感
- 預留 placeholder 文字，標記 `<!-- REPLACE: 品牌故事內容 -->`

### Section 3: 商品展示區

- 標題：「本季劇目」或「演出中」
- Grid 佈局：桌面 3 欄 / 平板 2 欄 / 手機 1 欄
- 每張商品卡片包含：
  - 商品圖片（正方形，先用 placeholder）
  - 商品名稱
  - 系列標籤（如「城市系列」「劇作家系列」）
  - 簡短風味描述（一行）
  - 價格
  - 「加入選購」按鈕
- 卡片樣式：深灰底 (#2A2A2A)、白色細邊框、hover 時邊框變亮
- 商品數據用 JavaScript 陣列管理，方便之後修改：

```javascript
const products = [
  {
    id: 1,
    name: "商品名稱",
    series: "系列名稱",
    flavor: "風味描述",
    roast: "中焙",
    origin: "產區",
    price: 180,
    image: "placeholder.jpg",
    description: "詳細描述..."
  },
  // ... 預設放入 6 款示範商品
];
```

### Section 4: 商品詳情 Modal

- 點擊商品卡片時彈出 Modal（不跳轉頁面）
- Modal 內容：
  - 大圖展示
  - 商品完整名稱與系列
  - 風味筆記（可用雷達圖或標籤呈現）
  - 烘焙度指示器（視覺化 — 如漸層條）
  - 產區資訊
  - 商品故事（2-3 句）
  - 規格與價格
  - 「加入選購」按鈕
- Modal 背景：半透明黑色遮罩，模擬劇場暗場效果
- 關閉方式：X 按鈕 + 點擊遮罩 + ESC 鍵

### Section 5: 訂購流程說明區

- 標題：「如何帶走你的故事」
- 3-4 步驟的水平流程圖（手機改為垂直）
- 步驟範例：
  1. 🎭 選擇你的劇目（挑選咖啡）
  2. 📝 填寫觀眾資訊（填寫訂購表單）
  3. 🏪 7-11 取件付款（貨到門市，取貨付款）
- 每個步驟用圓形 icon + 標題 + 一句說明
- 步驟之間用細線連接

### Section 6: 訂購表單區

- 標題：「預約你的座位」
- 兩種實作方式（請都準備，我選擇用哪個）：

**方案 A：嵌入式 Google Form**

- 用 iframe 嵌入，自訂外框樣式融合設計
- 預留 iframe src 替換標記

**方案 B：原生表單 + Google Apps Script**

- 自行設計表單 UI，與整體風格統一
- 表單欄位：
  - 姓名（必填）
  - 手機號碼（必填，格式驗證）
  - Email（選填）
  - 7-11 取貨門市（文字輸入 — 縣市+門市名稱）
  - 商品選擇（從已選購商品帶入，或 checkbox 多選）
  - 數量
  - 備註（選填）
  - 訂單總金額（自動計算顯示）
- 提交後送至 Google Sheets（透過 Google Apps Script Web App）
- 提交成功顯示確認訊息：「您的座位已保留！我們將盡快為您安排演出。」

### Section 7: FAQ 常見問題區

- 標題：「幕後花絮」
- 手風琴 (Accordion) 展開/收合互動
- 預設 5-6 題 FAQ（先用以下假內容）：
  - 如何付款？
  - 運費怎麼算？
  - 多久會到貨？
  - 可以退換貨嗎？
  - 咖啡豆保存方式？
  - 如何聯繫客服？
- 展開動畫要平滑

### Section 8: 顧客評價區

- 標題：「觀眾迴響」
- 水平滑動 Carousel / 或 Grid 呈現
- 每則評價：大引號裝飾 + 評價文字 + 顧客名稱（化名）+ 星等
- 先放入 4-6 則示範評價
- 資料一樣用 JS 陣列管理

### Section 9: Instagram / 社群連結區

- 標題：「追蹤我們的日常」
- 模擬 Instagram grid（2 排 x 3 張 = 6 張圖）
- 圖片先用 placeholder
- 點擊圖片跳轉 Instagram（target="_blank"）
- 下方放社群 icon 連結列（Instagram / LINE / Facebook）
- 預留連結替換標記

### Section 10: Footer

- 品牌名稱 + 一句品牌標語
- 聯絡資訊：Email / LINE 官方帳號 / 電話
- 營業時間（如有）
- 社群 icon 連結（同 Section 9，重複放置方便使用）
- 版權聲明：© 2025 咖啡小劇場 All Rights Reserved.
- 整體風格：極簡，細線分隔上方內容

## 🧩 全域功能需求

### Navigation

- 固定頂部導覽列（sticky）
- 背景初始透明，滾動後轉為半透明黑
- 左側：品牌名稱（文字 logo）
- 右側：各區塊錨點連結（平滑滾動 scroll-behavior: smooth）
- 手機版：漢堡選單，展開時全螢幕黑色覆蓋（劇場暗場感）

### 簡易購物車功能

- 不需要完整購物車系統
- 導覽列右上角「已選 N 件」提示
- 點擊展開側邊欄（Slide-in Panel），顯示已選商品、數量調整、總金額
- 「前往訂購」按鈕滾動至訂購表單區
- 資料存在 sessionStorage，重新整理不遺失

### 滾動動畫

- 使用 Intersection Observer API
- 各區塊進場時：淡入 + 微上移（fade-in-up）
- 動畫觸發閾值：元素進入視窗 20% 時觸發
- 動畫只觸發一次

### 效能要求

- 圖片使用 lazy loading
- CSS 動畫優先使用 transform 和 opacity（GPU 加速）
- 首屏載入時間目標 < 2 秒
- Lighthouse 效能分數目標 > 90

## 📌 重要規則

1. **每完成一個開發階段，先停下來展示進度，等我確認後再繼續下一步。**
2. 所有文字內容先用假文/範例填充，標記清楚 `<!-- REPLACE -->` 方便之後替換。
3. 所有圖片先用 placeholder（可用 https://placehold.co 或純色塊）。

7. 注意無障礙性（a11y）：語義化 HTML、適當的 aria 標籤、足夠的顏色對比度。
8. 用繁體中文回覆所有說明與討論。
